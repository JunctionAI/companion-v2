# companion

An open-source AI health coach that lives in your Telegram.

Reads your bloodwork, tracks your metrics, gives you a daily protocol. You own everything — your API key, your data, your bot.

## Quick Start (5 minutes)

```bash
git clone https://github.com/JunctionAI/companion-v2.git
cd companion-v2
npm install
npm run setup
npm run dev
```

The setup wizard walks you through:
1. Anthropic API key (get one at [console.anthropic.com](https://console.anthropic.com))
2. Telegram bot token (create via [@BotFather](https://t.me/botfather))
3. Your Telegram user ID (get via [@userinfobot](https://t.me/userinfobot))
4. Basic health info (name, age, goals)

Then message your bot. That's it.

## What It Does

**Daily check-ins** — Morning and evening protocols via Telegram. Tracks sleep, energy, mood, training, nutrition.

**Blood test interpretation** — Send a photo of your blood test results. Gets parsed by Claude Vision, stored, and interpreted in plain language with actionable recommendations.

**Health metrics** — Log data points throughout the day:
```
/log sleep 7.5
/log weight 82
/log mood 7
/log hrv 45
/data
```

**Permanent memory** — Learns about you over time. Every conversation is processed for facts (Claude Haiku, ~$0.001/extraction). Cross-session memory means your companion knows your history, patterns, and preferences.

**Mechanism-first coaching** — Explains the WHY, not just the WHAT. Evidence-based, never pseudoscientific.

**Self-onboarding** — New users message the bot and get walked through a questionnaire. Agent files generated automatically.

## Architecture

```
companion-v2/
  agents/           # Agent identity files (AGENT.md, skills, prompts, state)
    _template/      # Template for new agents (used by setup + onboarding)
    apex/           # Example agent
  config/
    agents.json     # Agent registry (generated by setup)
  data/
    user_memory.db  # SQLite — facts, messages, summaries, metrics, biomarkers
  src/
    config/         # Environment + agent config
    transport/      # Telegram polling + message dispatch
    modules/
      brain/        # File-based prompt assembly (3-block caching)
      memory/       # SQLite fact store + session history
      biomarkers/   # Blood test parsing via Claude Vision
      metrics/      # Health data logging (/log commands)
      onboarding/   # First-message questionnaire
      learning/     # Post-response marker extraction
      escalation/   # Safety tier system
      llm/          # Anthropic SDK with prompt caching
    workers/        # Scheduler + background fact extraction
```

**Prompt caching** — Three-block system for Anthropic's prompt caching:
- Static block (cached): Agent identity, health framework, training files
- Semi-static block (cached): User facts, plans, biomarkers, metrics
- Dynamic block (uncached): Date/time, recent session logs, context

This cuts API costs by ~90% for repeat conversations.

## You Own Everything

- **Your API key** — Anthropic calls go directly from your machine
- **Your data** — SQLite database stays on your machine
- **Your bot** — You created it, you control it
- **No cloud dependency** — Neo4j, Supabase, Redis all optional
- **Cost** — ~$1-3/month in API calls for daily use

## Commands

| Command | What it does |
|---------|-------------|
| `/log sleep 7.5` | Log a health metric |
| `/log mood 7` | Log mood (1-10) |
| `/data` | Show recent metrics |
| `/memory` | See what the bot remembers about you |
| `/forget <text>` | Delete matching memories |
| `/forget all` | Delete all memories |

**Available metrics:** sleep, hrv, weight, mood, energy, focus, steps, water, protein, calories

## Adding More Users

Each user gets their own Telegram group and agent folder:

1. Create a new Telegram group, add the bot
2. Have the user send a message in the group
3. The bot auto-detects the new chat and starts onboarding
4. Agent files generated from template, schedules registered

Or manually: edit `config/agents.json` and create an agent folder from `agents/_template/`.

## Customizing Your Agent

Edit files in `agents/<name>/`:
- `AGENT.md` — Identity, personality, user profile, protocols
- `skills/*.md` — Domain knowledge (nutrition, training, sleep)
- `prompts/*.md` — Scheduled task prompts (morning_checkin, evening_checkin)
- `state/CURRENT_PLAN.md` — Active health plan
- `state/CONTEXT.md` — Rolling context log (auto-updated)
- `training/*.md` — Deep domain training (evidence frameworks)

## Environment Variables

**Required:**
- `ANTHROPIC_API_KEY` — Your Anthropic API key
- `TELEGRAM_BOT_TOKEN` — Your Telegram bot token
- `TELEGRAM_OWNER_ID` — Your Telegram user ID

**Optional:**
- `TIMEZONE` — Default: UTC
- `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `SUPABASE_ANON_KEY` — Web API (if needed)
- `NEO4J_URI`, `NEO4J_PASSWORD` — Graph memory (adds to SQLite)
- `REDIS_URL` — BullMQ job queue
- `OPENAI_API_KEY` — Voice message transcription (Whisper)

## Deploying

**Railway / Render / Fly.io:**
```bash
npm run build
npm start
```

**Docker:**
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY . .
RUN npm ci && npm run build
CMD ["npm", "start"]
```
